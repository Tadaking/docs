# スキル：プラットフォームタブ追加

## 役割

iOS または Android の Figma ファイルが用意できたとき、
既存のコンポーネントガイドラインに該当プラットフォームのタブを追加する。

---

## インプット

| 項目 | 必須 | 説明 |
|---|---|---|
| Figma URL または node ID | ✅ | iOS または Android の対象コンポーネント |
| 対象プラットフォーム | ✅ | `ios` / `android` |
| 既存MDXファイルパス | ✅ | 追加先のコンポーネントガイドライン |

---

## 実行手順

### Step 1: 既存ファイルの確認

- 対象MDXファイルを読み込み、追加対象のプラットフォームタブが既に存在しないか確認する
- 存在する場合は上書きではなく差分更新を行う

### Step 2: Figmaクエリ

`generate-component-doc.md` の Step 1〜4 をプラットフォーム固有の情報に限定して実行する：

- `get_design_context` → バリアント・構成要素
- `get_variable_defs` → カラー・サイズトークン
- `get_screenshot` → プラットフォーム固有のビジュアル

### Step 3: タブの生成・追加

CLAUDE.md の `#### Web / iOS / Android タブの必須セクション` に従い、以下を生成する：
```
1. プラットフォームに特化した使い方やベストプラクティスとDo, Don'tなど
2. カラートークン
3. サイズ・スペーシング
4. インタラクションステート
```

生成したタブを既存MDXの `<Tabs>` 内に追加する。

### Step 4: Overview タブの確認

Overview タブは原則変更しない。
ただしプラットフォーム固有の重要な差異がある場合は、`<!-- NOTE: -->` コメントを追記するにとどめる。

---

## フォールバック

| 状況 | 対処 |
|---|---|
| 対象タブが既に存在する | 差分のみ更新し、既存コンテンツを保持する |
| トークンが取得できない | テーブルをプレースホルダーで生成し `（要確認）` と記載 |
