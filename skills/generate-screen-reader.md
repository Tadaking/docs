# スキル：スクリーンリーダー仕様生成

## 役割

コンポーネントのアクセシビリティセクション（ARIA / VoiceOver / TalkBack）を生成し、
コンポーネントガイドラインの `## アクセシビリティ` セクションに挿入する。

---

## インプット

| 項目 | 必須 | 説明 |
|---|---|---|
| Figma URL または node ID | ✅ | 対象コンポーネント |
| 対象プラットフォーム | ✅ | `web` / `ios` / `android` / `all` |
| 出力先MDXファイルパス | ✅ | 挿入先のコンポーネントガイドライン |

---

## 実行手順

### Step 1: コンポーネント情報の取得
```
get_design_context(nodeId)
```

- コンポーネントの役割（ボタン・入力フィールド・ナビゲーション等）を判定する
- インタラクティブな要素かどうかを確認する

### Step 2: アクセシビリティ仕様の生成

コンポーネントの種別に応じて以下を生成する：

#### Web（ARIA）
```md
| 属性 | 値 | 説明 |
|---|---|---|
| `role` | （適切なrole値） | （説明） |
| `aria-label` | （推奨ラベル） | （説明） |
| `aria-disabled` | `true` / `false` | disabled状態で付与 |
```

キーボード操作：
```md
| キー | 動作 |
|---|---|
| `Enter` / `Space` | アクティベート |
| `Tab` | フォーカス移動 |
```

#### iOS（VoiceOver）
```md
| プロパティ | 値 |
|---|---|
| Accessibility Label | （推奨テキスト） |
| Accessibility Trait | （適切なTrait） |
| Accessibility Hint | （操作の説明） |
```

#### Android（TalkBack）
```md
| プロパティ | 値 |
|---|---|
| contentDescription | （推奨テキスト） |
| importantForAccessibility | `yes` |
```

### Step 3: コントラスト要件の記載

CLAUDE.md のアクセシビリティ基準に従い記載する：
```md
| テキスト種別 | 最低基準 | 使用トークンの組み合わせ |
|---|---|---|
| UIテキスト | Lc 60以上 | （トークン名） on （トークン名） |
| 本文テキスト | Lc 75以上 | （トークン名） on （トークン名） |
```

### Step 4: MDXへの挿入

対象MDXファイルの `## アクセシビリティ` セクションを生成した内容で更新する。

---

## フォールバック

| 状況 | 対処 |
|---|---|
| コンポーネント種別が判定できない | 汎用テンプレートを挿入し `<!-- TODO: role・aria属性を確認 -->` を追記 |
| プラットフォーム固有情報が不足 | 取得できた範囲で生成し、不足部分に `（要確認）` と記載 |
